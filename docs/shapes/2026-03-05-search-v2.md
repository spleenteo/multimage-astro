# Shape: Search v2

**Status**: in-progress
**Date**: 2026-03-05
**Appetite**: medium (2 sessions)

## Problem

`/cerca` works but is limited:

- No filter by collection (`collana`) — users can't narrow to a specific series.
- No sorting — results appear in Dato relevance order only.
- The page is 475 lines with ~200 lines of inline CSS — hard to maintain and extend.
- The archived-book badge was just added but the visual treatment and UX could be improved.
- No "no results" state with suggestions.
- Missing runtime assertions — if `data-*` config is absent the page fails silently.

## Solution sketch

- **Refactor `/cerca` styles**: extract inline CSS to `src/pages/cerca/_style.module.css`.
- **Add collection filter**: extend `search-page.client.ts` to support a `collana` filter using Dato Site Search `filter` param. Requires adding collection name to the search index.
- **Add CH2 runtime guard**: throw a descriptive error when required `data-*` attributes are missing.
- **Improve no-results UX**: show a helpful message with suggestions when the result set is empty.
- **Stabilize archived badge**: ensure the badge renders correctly in all result card variants.

## Rabbit holes

- Collection filtering depends on the DatoCMS search index including collection name — verify before building.
- Sorting may not be supported by Dato Site Search without a custom index configuration.

## No-gos

- No custom Elasticsearch or Algolia migration.
- No full-text content re-indexing.
