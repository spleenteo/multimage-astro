# Shapes & Slices

This folder tracks **shaped work** for Multimage — problem definitions and solution sketches used to decide what to build next.

## What is a Shape?

A shape is a scoped proposal for a piece of work. It defines:
- **The problem** — what is broken, missing, or painful
- **The appetite** — how much time/effort is acceptable
- **The solution sketch** — the approach at a fat-marker level (no implementation details)
- **Rabbit holes** — known complexity to avoid or de-risk
- **No-gos** — explicit scope boundaries
- **Slices** — the concrete tasks to execute once the shape is approved

Shapes live here before work starts. Once a shape is in progress, its slices move to `docs/TODO.md`.

## Status values

- `idea` — rough concept, not yet shaped
- `shaping` — being refined, still open questions
- `ready` — approved and ready to start
- `in-progress` — at least one slice is being worked on
- `done` — all slices completed

## File naming

```
YYYY-MM-DD-short-name.md
```

Example: `2026-03-05-security-baseline.md`
